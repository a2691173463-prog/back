# 简历模板第二至第四阶段实施说明

## 实施范围

### 第二阶段：模板产品化

- 模板广场保留搜索、分类、排序、详情预览和下载能力。
- “使用模板”升级为“在线制作”，模板分类会决定简历强调色和展示重点。
- 管理员新增模板时，缩略图和外部下载地址改为可选；没有外链时使用系统内置 A4 预览和在线导出。

### 第三阶段：在线制作与草稿

- 按基本信息、教育经历、项目经历、实习经历、技能荣誉分区编辑。
- 右侧实时生成 A4 简历，表单修改会立即反映到预览。
- 浏览器本地自动保存，不登录也不会因误关闭丢失当前内容。
- 登录后可将草稿保存到 MySQL，并按用户 ID 隔离数据。
- 同一模板可以从最新的云端草稿恢复。

### 第四阶段：智能优化与交付

- AI 可以分别优化个人优势、项目经历和实习经历。
- 调用路径为 `Vue -> Spring Boot -> FastAPI -> 大模型`。
- Python 提示词明确禁止虚构经历、技术和量化数据。
- ATS 完成度检查覆盖联系方式、求职方向、教育、项目细节、量化成果和技能匹配。
- 使用浏览器打印能力导出标准 A4 PDF。

## 数据库迁移

现有数据库需要执行一次：

```sql
SOURCE D:/简历项目/back/sql/resume_draft.sql;
```

也可以直接在数据库客户端执行 [resume_draft.sql](../back/sql/resume_draft.sql)。全新环境执行 `back/sql/schema.sql` 时会自动创建该表。

## 新增接口

| 方法 | 地址 | 用途 |
| --- | --- | --- |
| GET | `/api/resume-drafts` | 查询当前用户云端草稿 |
| GET | `/api/resume-drafts/{id}` | 查询当前用户的指定草稿 |
| POST | `/api/resume-drafts` | 新建或更新草稿 |
| DELETE | `/api/resume-drafts/{id}` | 删除当前用户草稿 |
| POST | `/api/templates/optimize` | 鉴权后转发简历内容优化请求 |
| POST | Python `/resume/optimize` | 调用模型改写指定简历段落 |

## 验证清单

1. 执行数据库迁移并重启 Java 后端。
2. 重启 Python Agent 服务，使 `/resume/optimize` 生效。
3. 进入模板广场，点击“在线制作”。
4. 编辑姓名、教育和项目，确认 A4 预览同步更新。
5. 登录后测试保存云端、恢复云端和 AI 优化。
6. 点击“导出 PDF”，在浏览器打印窗口选择“另存为 PDF”。
